# DESC stellarator B-field benchmark

This benchmark compares the NUFFT Biot-Savart field against the solved DESC
stellarator equilibrium field inside the plasma. The physical comparison is

```text
B_model = B_NUFFT(plasma J) + B_external/coils
```

The benchmark is configuration-driven: each stellarator case lives in a small
JSON file under [`configurations/`](configurations/), and `--config NAME`
selects it. The default `precise_QA` configuration is the validated working
case (good agreement at `N=64`). A `W7-X` configuration is also provided.
Explicit command-line flags always override the configuration file.

## Files

- `benchmark_desc_bfield.py`: the benchmark driver (one or more `N` per run).
- `submit_desc_bfield_scan.py`: submit one GPU Slurm job per `N`.
- `aggregate_error_scan.py`: combine the per-`N` job results into one
  `error_vs_N.png` convergence plot.
- `slurm_desc_bfield.sbatch`: standalone single-job script (all `N` in one run).
- `bench_config.py`: shared configuration loader.
- `configurations/`: the available stellarator configurations.

## Configurations

Each configuration is a subdirectory of `configurations/` containing the DESC
h5 files for that case. Select one with `--config <name>`.

```
configurations/precise_QA/eq.h5        ← equilibrium (required)
configurations/precise_QA/coils.h5     ← coil field  (optional)
configurations/W7-X/eq.h5
configurations/W7-X/coils.h5
```

To add a case, create a new subdirectory and drop in `eq.h5` / `coils.h5`.
See [`configurations/README.md`](configurations/README.md) for copy commands.

## Single run

```bash
python benchmarks/benchmark_desc_bfield.py \
  --config precise_QA \
  --desc-root /path/to/DESC \
  --outdir results/desc_bfield/precise_QA \
  --n-values 64 128 256 512 \
  --jax-platform cuda
```

The equilibrium, coils, source model, padding, `eps`, and the angular grid all
come from the configuration; pass the matching flag to override any of them
(e.g. `--n-rho 32`, `--eq-file ...`, `--coil-file ...`).

If the DESC checkout is a sibling of this `nufft` directory, `--desc-root auto`
will find it. Use `--desc-root installed` to force the active Python
environment's installed DESC package.

Use `--jax-platform cuda` only inside a batch or interactive allocation where
the job can see an NVIDIA GPU. On a login node or CPU-only smoke test, use
`--jax-platform cpu`, or omit the flag to let JAX choose automatically. The
legacy `--jax-platform gpu` flag is accepted as a CUDA alias.

## Scan as separate Slurm jobs

```bash
python benchmarks/submit_desc_bfield_scan.py --config precise_QA
```

This submits one job per `N` and writes results under
`results/desc_bfield/<config>/N64`, `N128`, `N256`, `N512`. Jobs are skipped if
their output files already exist or if a matching job name is pending/running.

For a different case (with a paired free-boundary equilibrium and coils):

```bash
python benchmarks/submit_desc_bfield_scan.py \
  --config W7-X \
  --coil-file /path/to/coils-optimized.h5
```

Restrict to a single `N` with `--n-values 64`.

## Aggregating the convergence plot

Each separate Slurm job runs a single `N`, so its own `error_vs_N.png` is not
produced (a single point is not a scan). After the jobs finish, combine every
`N*/metrics.csv` into one convergence plot summed over all tested `N`:

```bash
python benchmarks/aggregate_error_scan.py --base-dir results/desc_bfield/precise_QA
```

This writes `error_vs_N.png` and `error_vs_N.csv` at the base of the
configuration's results directory. (When a single run covers multiple `N` in
one process, that run still writes its own multi-point `error_vs_N.png`.)

## Outputs

Per `N` directory:

- `metrics.csv`: timing and error metrics for that `Nx=Ny=Nz`.
- `run_metadata.json`: DESC/JAX/device metadata, the configuration, and grid settings.
- `desc_bfield_sample_N*.npz`: sampled DESC and NUFFT fields used for plots.
- `quiver_zeta*_N*.png`: in-plane `BR,BZ` quiver plots over DESC flux surfaces.
- `components_zeta*_N*.png`: `BR`, `BZ`, and `Bphi` cross-section comparisons.

Per configuration (after aggregation):

- `error_vs_N.png` / `error_vs_N.csv`: relative error versus Fourier grid size.

The B evaluation grid spans flux surfaces from the first interior shell out to
the LCFS (`rho=1`), so the plotted/compared field fills the plasma
cross-section exactly up to the boundary.

## Interpretation notes

DESC total `B` is not generated by the plasma volume current `J` alone; it also
contains the vacuum/external field. The benchmark therefore requires a paired
coil/external field by default. The optional `--source-model boundary` path uses
DESC's equivalent LCFS sheet current as a virtual-casing diagnostic; it can be
combined with `--coil-file` for an HBT-style `B_boundary + B_coils` check, but
the scan defaults to `--source-model volume` for the inside-plasma benchmark.
Pass `--allow-missing-coils` for a plasma-current-only diagnostic when no coil
field is available.

Weight/NFP note: DESC `LinearGrid.weights` are already normalized to the full
angular domain. When NUFFT source nodes are explicitly replicated over `eq.NFP`
field periods, each replicated point receives `weight / eq.NFP`, preserving the
DESC integral while giving NUFFT the full torus geometry.
